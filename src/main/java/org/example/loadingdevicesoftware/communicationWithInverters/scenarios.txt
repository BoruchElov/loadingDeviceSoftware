SET_SCENARO_1(P1,P2,P3,P4,P5,P6,P7,P8) - настройка сценария выдачи линейного тока
P1 - Ток
P2 - Фаза
P3 - Время
P4 - Частота
P5 - Контакт 1 (0/1 - нормально-замкнутый/нормально-разомкнутый)
P6 - Контакт 1 (1/0 - сухой/мокрый)
P7 - Контакт 2 (0/1 - нормально-замкнутый/нормально-разомкнутый)
P8 - Контакт 2 (1/0 - сухой/мокрый)


SET_SCENARO_2(P1,P2,P3,P4,P5,P6,P7,P8) - настройка сценария выдачи импульсного тока
P1 - Ток
P2 - Фаза
P3 - Время
P4 - Частота
P5 - Контакт 1 (0/1/2 - нормально-замкнутый/нормально-разомкнутый/контакт отключен)
P6 - Контакт 1 (1/0 - сухой/мокрый)
P7 - Контакт 2 (0/1/2 - нормально-замкнутый/нормально-разомкнутый/контакт отключен)
P8 - Контакт 2 (1/0 - сухой/мокрый)


Вопрос 1: Обсуждалось, что на форме будет только одна кнопка для задания типа контакта (сухой/мокрый). Но в аргументах
функции SET_SCENARO_1 тип задаётся для каждого контакта. Как в конечном итоге задавать тип контакта?

Ответ: тип контакта задается одновременно для обоих контактов

Вопрос 2: Обсуждалось, что на форме будет кнопка для условия отключения. Но в аргументах функции SET_SCENARO_1 этого
свойства нет. Куда оно идёт и как его использовать?

Ответ: данное свойство идет в качестве аргумента для функции задания сценария (1/0)

Вопрос 3: Правильно ли описаны аргументы для задания сценария?

Ответ: Да, но у каждой функции добавляются ещё 2 аргумента - P8 и P9

Вопрос 4: Где используется параметр частоты? Или это не параметр, а информация с датчика?

Ответ: Этот параметр передаётся в функцию

Условие отключения:
1) t/сраб - контакты заблокированы, сценарий завершается только по истечении времени (в этом случае не нужно учитывать
время срабатывания)
2) Контакт - сценарий завершается по сигналу контактов (в этом случае сценарий успешен), если сценарий завершился по
таймауту, а не по контактам, то сценарий неуспешен

Логика добавления ошибок, три положения контактов, фаза для амперметра, второе время срабатывания, статус завершения
сценария
